# 项目背景

## 项目概述

这个项目是一个面向 Agent 调用 ComfyUI 工作流的 MVP。

当前目标**不是**在第一阶段就做成一个完整的通用工作流平台，  
而是先从**一个固定工作流**开始，把执行链路跑通，并验证整体架构方向是否成立。

当前预期使用场景包括：

- OpenClaw / 类 Claw Agent
- ComfyUI 整合包环境
- 以 Skill 形式进行接入
- 未来在有需要时，可基于同一套核心能力扩展到其他接入方式

## 当前基础仓库

当前实现基础是已有仓库：

- `MieMieeeee/run_comfyui_workflow`

这个仓库已经具备：

- 工作流执行封装
- 示例运行脚本
- 内置的 `z_image_turbo` 工作流
- 简化版 ComfyUI API 调用能力

## 当前第一阶段工作流选择

第一阶段 MVP 固定使用仓库自带工作流：

- `workflows/z_image_turbo.json`

参考示例脚本为：

- `run/get_zimage.py`

## 为什么选择这个工作流

之所以把这个工作流作为第一阶段试点，是因为：

1. 它已经存在于当前仓库中
2. 它已经有现成的参考脚本
3. 在 MVP 场景下它只需要文本输入
4. 它通过 `Save Image` 节点清晰地产出图片
5. 它避免了第一阶段就处理文件上传的复杂度

## 已确认的工作流事实

根据当前仓库文件，已经确认以下关键节点信息：

- 正向提示词节点标题：
  - `CLIP Text Encode (Positive Prompt)`

- 负向提示词节点标题：
  - `CLIP Text Encode (Negative Prompt)`

- 尺寸节点标题：
  - `EmptySD3LatentImage`

- 输出节点标题：
  - `Save Image`

- 采样节点标题：
  - `KSampler`

当前示例脚本展示了一条可工作的执行路径：

- 加载 `workflows/z_image_turbo.json`
- 设置正向提示词
- 可选设置宽高
- 调用 `api.queue_and_wait_images(wf, output_node_title="Save Image")`
- 将返回的图片字节保存到本地结果目录

## MVP 定位

这是一个**面向 Skill 的 MVP**，不是最终平台形态。

在当前阶段，我们只需要产出一个最小可复用能力，能力形式可以理解为：

```python
text_to_image(prompt: str)
```

实现应尽量保持简单，同时为未来扩展预留基本空间。
