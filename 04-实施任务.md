# 实施任务

## 你的任务

请在仓库 `MieMieeeee/run_comfyui_workflow` 中，为固定 workflow `z_image_turbo` 实现一条最小可复用的执行链路。

## 必读文件

在做任何修改前，请至少阅读以下文件：

- `workflows/z_image_turbo.json`
- `run/get_zimage.py`
- `comfy_api_simplified` 目录下相关封装

## 需要产出的结果

请完成以下内容：

### 1. 最小代码结构方案
请提出一版最小化的目录/文件结构方案。

可以类似：

- `services/execute_z_image_turbo.py`
- `models/result.py`
- `skill/text_to_image.py`

如果你认为更轻量的结构更合理，也可以采用更简单方案，但请简要说明原因。

### 2. 一个可复用执行函数
请实现一个等价于下面形式的函数：

```python
execute_z_image_turbo(prompt: str) -> dict
```

这个函数应完成：

- 加载固定 workflow
- 设置正向提示词
- 如有需要，设置固定宽高
- 通过现有 wrapper 执行
- 将结果保存到本地
- 返回结构化结果

### 3. 一个更高层入口函数
可以选择性提供一个更高层函数，例如：

```python
text_to_image(prompt: str) -> dict
```

这个函数可以只是对 workflow 专属执行函数的薄封装。

### 4. 统一结果结构
请定义清晰的结果结构。

成功结果至少包含：

- `success`
- `workflow_id`
- `status`
- `outputs`

可选但推荐包含：

- `job_id`
- `metadata`
- `error`

### 5. 一个最小使用示例
请给出如何调用新函数的使用示例。

## 设计约束

请严格遵守以下约束：

- 实现尽量小
- 实现尽量清晰
- 优先从当前示例脚本演化，而不是完全重写
- 如果函数足够，不要为了形式强行上很多类
- 不要实现 MCP
- 不要实现多 workflow 支持
- 不要过度工程化
- 可以保留未来扩展空间，但这不是第一优先级

- 执行失败
