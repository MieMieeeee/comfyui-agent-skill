# 目标与范围

## 核心目标

基于固定工作流：

- `workflows/z_image_turbo.json`

实现一个最小可复用的文本生图能力。

当前 MVP 只允许一个外部输入参数：

- `prompt`

其余参数在第一阶段全部保持固定。

## 预期能力形式

期望得到的高层能力概念上类似：

```python
result = text_to_image(prompt="...")
```

返回值应是一个统一的结构化结果对象。

## 第一阶段范围内

第一阶段只包含以下内容：

1. 一个固定工作流：`z_image_turbo`
2. 一个外部输入参数：`prompt`
3. 一种输出类型：图片
4. 一个小型可复用执行模块
5. 一个统一结果结构
6. 可选提供一个简单 CLI 或入口函数
7. 如果实现成本不高，可补充基础服务可用性检查

## 第一阶段范围外

第一阶段**不包含**以下内容：

- MCP
- 多工作流编排
- 自动发现 workflow
- 自动分析 workflow
- 图生图
- 图生视频
- 文生视频
- 用户自定义负向提示词
- 用户自定义 seed
- 用户自定义采样参数
- 用户自定义 steps / cfg
- 大规模重构
- 复杂插件或模型管理
- 完整平台化架构

## 当前固定运行假设

第一阶段可以接受以下参数固定：

- workflow 文件：`workflows/z_image_turbo.json`
- 输出节点：`Save Image`
- width：`832`
- height：`1280`
- 正向提示词来自外部输入
- 负向提示词沿用 workflow 默认值
- seed / steps / cfg 沿用 workflow 默认值，除非现有脚本已明确设置

## 成功标准

如果满足以下条件，则认为 MVP 成功：

1. 调用方可以传入一个 prompt
2. 代码可以成功运行 `z_image_turbo` 工作流
3. 输出图片可以成功保存到本地
4. 调用方能收到清晰的结构化结果
5. 实现方式比当前单一脚本更清晰、可复用
6. 代码总体保持小而易懂
